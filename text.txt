# Audit Table

The document only specifies storing hashed UIN, creation timestamp, and additional metadata, so I'd extend it slightly for better traceability while keeping the required fields. 

```sql
CREATE TABLE deletion_audit(

id UUID PRIMARY KEY,

hashed_uin VARCHAR(256),

request_time TIMESTAMP,

status VARCHAR(50),

registration_status VARCHAR(50),

idrepo_status VARCHAR(50),

ida_status VARCHAR(50),

resident_status VARCHAR(50),

self_registration_status VARCHAR(50),

remarks TEXT

);
```

# Error Handling

The document says **never stop** if one module fails; continue deleting the rest and report aggregated results. 

So

```java
try{

registration.delete();

}catch(Exception e){

status.setRegistration(FAILED);

}
```

Continue

```java
try{

idRepo.delete();

}catch(...){}
```

Continue

until all modules finish.

## My suggestion

Don't try to build the entire service at once. It's a fairly large backend that coordinates multiple systems. The most efficient approach is to build it in phases:

1. Spring Boot project setup
2. Database configuration
3. eSignet authentication (`/token` and `/userinfo`)
4. Deletion orchestrator
5. Registration Processor deletion
6. ID Repository deletion
7. IDA deletion
8. Resident Portal deletion
9. Audit logging
10. Dockerization and testing

That way, each phase is testable before moving to the next.
